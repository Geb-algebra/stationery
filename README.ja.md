# Stationery

Stationery is a lightweight toolkit for AI-assisted coding.  
Each tool is small, simple, and flexible — you can start using it anytime,  
and step away just as easily.

From early prototyping to large-scale enterprise development,  
Stationery supports your workflow without taking control.  
It adapts to *your* way of building.

## モチベーション

Stationeryは、vibe-codingの手軽さとSpec-Driven Developmentの堅実さのいいとこ取りを目指したツールキットです。

思うがままにAIと対話してコードを書いていくVibe Codingは、とても簡単にアイデアを形にできます。
でも、それは最初のうちだけ。プロジェクトが大きくなるにつれて、意図が曖昧になり、仕様がずれて、気づいたらメンテナンス不可能になってしまいます。

仕様書駆動開発はその問題を解決するために生まれた、AIと協調しながらも破綻せずに開発を続けていくための手法です。
Spec KitやOpenSpecといった仕様書駆動開発ツールを実際に使ってみると、この手法は確かに強力であることがわかります。

しかし、これらのツールは少し重たいと感じるときがあります。例えば

- **プロジェクト全体がそのツールにロックインされる**: そのツールが仕様書のフォーマットをすべて規定するため、一度始めると他のツールに移れません。
- **開発プロセスが強制される**: 開発プロセスもそのツールが規定するため、あなたやプロジェクトチームの好みの開発スタイルをとれません。
- **既存プロジェクトへの導入はほぼ不可能**: すべてのコードに対応する仕様書を、時間と労力をかけて整備しない限り、本格的には使えません。 
- **膨大な仕様書をメンテナンスし続ける労力が大きい**: 自然言語はコードより曖昧であるため、開発を重ねると、仕様書と実装の乖離や仕様書間の矛盾が容易に起こります。 開発を続けるためには、そのような仕様書のミスがないか常に目視で確認し続ける必要があります。

Stationeryは、仕様書駆動開発の堅牢性をなるべく残しながら、このような「重さ」をなくしたツールが欲しいという思いから生まれました。

**プロジェクトをロックインしません。**
Stationeryのツールは、あなたのプロジェクトのどのリソースにも依存しません。どのリソースをどう使うか、すべてあなたが決められます。

**ツールのポリシーに合わせる必要はありません。**
Stationeryは開発プロセスを規定しません。好きなツールだけ使ってください。あなたのやり方に合わないツールは置いておきましょう。
ツールのやり方に合わせる必要はありません。ツールをあなたのやり方に合わせて使えます。

**準備なしで簡単に始められます。**
Stationery導入のためにコマンド・環境構築・情報整理・意思決定は必要ありません。
このリポジトリの中の好きなファイルを1つ、あなたのリポジトリに持っていくだけで使い始められます。

**やめたくなったら、しまうだけ。**
Stationeryのツール、その生成物、すべて永続化もメンテナンスも不要です。
使い終わったら、削除しても構いません。
必要なものだけを、必要なだけ使ってください。

Stationeryは開発の仕方を定義・強制・支配するフレームワークではありません。 ちょうど文房具のように、小さく、汎用的で、どんな場面にもあなたの思う使い方で活用できる、小さなツールの集まりです。
